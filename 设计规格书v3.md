## 一、数据库设计（4 张表）

### 表 1：memories（通用记忆）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| layer | TEXT | **core**(深层,永不过期) / **daily**(日常,5天过期) / **diary**(日记) |
| title | TEXT | 标题（必填） |
| content | TEXT | 内容正文 |
| tags | TEXT | 标签，逗号分隔 |
| mood | TEXT | 日记心情：happy/sad/calm/excited/anxious |
| meta | TEXT | JSON 扩展字段（未来健康记录、阅读笔记等） |
| source | TEXT | claude / user / system |
| created_at | TEXT | 创建时间 |
| updated_at | TEXT | 更新时间 |

#### 三层定义（也会写入 MCP 工具描述，让 Claude 知道怎么分层）

**core（深层记忆）— 永不过期**
- 存什么：长期不变的信息。用户身份、个人偏好、重要规则、记忆库使用规范、核心价值观/目标。
- 不存什么：具体事件、近期发生的事、会过时的信息。
- 写入频率：很少，只有确认需要长期记住的信息才写入。
- 示例：用户偏好（中文回复、简洁风格）、收工流程规则、各层使用规范、"每月15号固定还信用卡"这类永久性提醒。

**daily（日常记忆）— 5 天自动过期**
- 存什么：近几天发生的事、对话摘要、临时上下文、短期需要记住的信息。
- 不存什么：需要长期保留的内容（该放 core）、带感情的日记（该放 diary）、具体待办事项（该放 todos）。
- 写入频率：频繁，每天的收工总结、对话中的临时信息都放这里。
- 过期逻辑：创建超过 5 天后由定时任务自动删除。
- 示例：今天和老王讨论了项目用 React 方案、这周情绪比较低落需要注意。

**diary（日记）— 永久保留**
- 存什么：每天一篇带感情色彩的个人记录，带 mood 标签。
- 不存什么：流水账式的事件罗列（那是 daily 的事）、客观信息。
- 写入频率：每天最多一篇（收工流程时写入）。
- 示例：今天搭好了记忆库，特别有成就感。第一次用 wrangler CLI 比想象中顺利……（mood: happy）

#### 初始 core 记忆（搭建完成后建议写入的第一批深层记忆）

搭建完成后，建议通过前端页面或让 Claude 写入以下 core 记忆：

1. **记忆库使用规范**：各层定义（上面这段精简版）、写入原则、清理规则
2. **收工流程规则**：触发词、执行步骤（见本文档第八节）
3. **用户基本信息和偏好**：语言、风格、常用分类等
4. **其他长期规则**：你认为 Claude 每次都应该知道的信息

## 二、MCP 工具（3 个）

### read
- 通过 `type` 参数区分：memory / todo / rule / habit
- 支持筛选：keyword, date_from, date_to, layer, category, status
- 习惯返回最近 N 天的打卡状态和连续天数/完成率摘要
- 返回紧凑 JSON（去除 null 字段省 token）

### write
- 通过 `type` 参数区分操作对象
- 传 id → 更新，不传 → 创建
- type=todo 且 status=postponed 时：自动在 postponed_to 日期创建一条新 pending 记录
- type=habit 且带 checkin 参数时：追加到 log 字段，不创建新行

### delete
- 按 id 删单条
- 按 layer + days 批量清理（如 layer=daily, days=5 → 删 5 天前的日常记忆）

## 三、定时任务（Cron Triggers）

| 时间 | 动作 |
|------|------|
| 每天 03:00 UTC | 清理 5 天前的 daily 记忆 |
| 每天 03:05 UTC | 检查活跃循环规则，为未来 7 天生成 todos 实例（同 rule_id + 同日期不重复） |

## 四、前端页面

- **深色主题**，风格参考蛋壳方案截图
- **顶部标签**：全部 / 深层 / 日常 / 日记 / 待办 / 习惯
- **搜索栏**：关键词搜索
- **视图切换**：列表 / 日历
- **列表视图**：记忆卡片，带分层徽章、分类色块、状态标签
- **日历视图**：
  - 日记 → 心情色点（绿=happy, 蓝=sad, 灰=calm, 黄=excited, 粉=anxious）
  - 日程 → 分类色点（蓝=工作, 绿=生活, 紫=学习）
  - 点击日期 → 展开当天详情
- **习惯面板**：每个习惯一行，最近 7 天的打卡格子（绿=全完成, 黄=部分, 灰=未做）
- **添加/编辑弹窗**：根据选择的类型动态显示对应字段
- **右下角 "+" 按钮**：快速添加

## 六、关键行为规则汇总

| 规则 | 说明 |
|------|------|
| daily 过期 | 5 天自动清理 |
| 待办推迟 | 原记录标记 postponed + postponed_to，自动在新日期创建新 pending |
| 循环实例生成 | 提前 7 天，同 rule_id + 同日期去重 |
| 习惯打卡 | 追加到 log 字段，不生成独立行 |
| 分类 | TEXT 字段，预设 工作/生活/学习，可随时新增 |
| 可扩展 | meta JSON 字段承载未来新功能（健康、阅读等） |
| 导出 | /api/export 一键 JSON，支持按时间范围 |
| 安全 | 可选 AUTH_TOKEN，URL 带 ?token=xxx |
| 多端 | 同一 URL 可添加到多个 Claude 账号 |

## 七、搭建步骤（不变）

1. 解压项目 → npm install
2. npx wrangler login
3. npx wrangler d1 create memory-hub-db → 复制 database_id 到 wrangler.toml
4. npm run db:init（执行建表）
5. npm run deploy（部署）
6. claude.ai → Settings → Connectors → Add custom → 填 URL/mcp
7. 新对话中启用连接器，开始使用

## 八、收工流程（写入 Claude 记忆的规则模板）

搭好后在 Claude Memory 中添加：

> 【收工流程规则】当用户说"收工""今天结束""晚安"等结束信号时，
> 自动执行记忆库收工流程：
> ①read(type=todo,status=pending,date=今天) 查看今日待办完成情况
> ②write(type=memory,layer=daily) 写今日总结
> ③write(type=memory,layer=diary,mood=...) 写日记
> ④delete(type=memory,layer=daily,days=5) 清理过期日常
